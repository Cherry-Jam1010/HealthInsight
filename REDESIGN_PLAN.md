# HealthInsight 产品重新设计方案

## 一、问题诊断

当前产品最核心的问题是：**产品定位不清，用户找不到"入口"**。

| 问题 | 现状 | 影响 |
|------|------|------|
| 功能堆砌 | 首页塞满了指标卡片、API 示例、风险因素、双周期对照等，没有优先级 | 用户进来不知道从哪开始 |
| 没有应用场景引导 | 一直是在展示"功能"，而不是在帮助用户"完成任务" | 用户不知道这个工具对他有什么用 |
| 导航混乱 | 三层嵌套下拉菜单，很多页面意义不明确 | 用户迷路，不知道去哪 |
| 缺乏核心操作流 | 没有"录入→分析→出报告"这样的完整路径 | 产品像一个展示柜，不像一个工具 |
| 报告功能太深 | 最重要的"生成报告"功能埋在了 Reports 页面里 | 用户找不到它 |

---

## 二、重新定位

**HealthInsight 是一个"心理健康风险识别工作台"，不是仪表盘展示站。**

核心使用场景：
1. **导入数据**（机构自己的样本，或生成演示数据）
2. **一键分析**（系统自动计算风险、找出重点人群、识别关键因素）
3. **生成报告**（一键生成面向管理/临床/研究团队的可读报告）

目标用户心智模型：

> "我有一个样本表，我想知道这里面哪些人风险更高，然后给领导写一份报告。"

---

## 三、新产品结构

### 导航设计（精简为 3 大区块）

```
┌─────────────────────────────────────────────┐
│  🏠 首页    📊 工作台    📋 报告中心         │
└─────────────────────────────────────────────┘
```

| 导航项 | 旧导航对应 | 定位 |
|--------|-----------|------|
| 首页 | 原 / | **产品介绍**：我是谁、有什么用、如何开始 |
| 工作台 | 原 /studio + /workbench | **核心应用**：数据导入 + 实时分析 |
| 报告中心 | 原 /reports | **输出交付**：生成和下载正式报告 |

**保留下来的辅助页面（入口从页脚或首页引导）：**
- **全球视野** (`/vision`)：国际参考，保持不变
- **使用指南** (`/guide`)：帮助文档，保持不变
- **API 文档** (`/docs`)：技术入口，保持不变

**删除的页面：**
- `/scenarios`（应用场景）→ 内容合并到首页
- `/examples`（落地实例）→ 内容合并到首页

## 四、首页重新设计

### 4.1 设计目标

首页需要回答三个问题：
1. **这是什么？** → 一句话定位
2. **对我有什么用？** → 典型使用场景
3. **怎么开始？** → 清晰的操作引导

### 4.2 首页结构

```
┌────────────────────────────────────────────────────────┐
│  [HI] HealthInsight                                    │
│                                                        │
│  ┌──────────────────────────────────────────────────┐  │
│  │  更快找到需要心理健康筛查支持的人群。              │  │
│  │                                                  │  │
│  │  把 PHQ-9 数据变成可直接交给机构的风险报告。       │  │
│  │                                                  │  │
│  │  [开始分析]  [查看报告示例]                       │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  ── 三步，完成一次风险分析 ──                          │
│                                                        │
│  ① 导入样本 → ② 查看分析结果 → ③ 生成报告              │
│                                                        │
│  ── 典型使用场景 ──                                    │
│                                                        │
│  [场景卡片] [场景卡片] [场景卡片]                       │
│                                                        │
│  ── 平台总览 ──                                        │
│                                                        │
│  当前数据样本量  |  高风险占比  |  覆盖地区             │
│                                                        │
│  [进入工作台]                                          │
└────────────────────────────────────────────────────────┘
```

### 4.3 关键改动

- **删除**：所有 API 预览、代码片段、JSON 输出
- **新增**：三步操作流程可视化（导入→分析→报告）
- **新增**：典型场景卡片（机构新筛查、区域基线摸排、资源规划等）
- **新增**：直接进入工作台的 CTA

## 五、工作台重新设计

### 5.1 设计目标

工作台是产品的"核心操作页"，用户在这里完成主要任务。

### 5.2 工作台结构

```
┌─────────────────────────────────────────────────────────┐
│  数据导入区域（简化）                                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  拖拽 CSV 或 粘贴数据 或 快速生成演示样本          │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  分析结果（自动计算，实时展示）                      │   │
│  │                                                   │   │
│  │  风险分布图  |  重点人群  |  关键风险因素            │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  [生成正式报告]                                         │
└─────────────────────────────────────────────────────────┘
```

### 5.3 关键改动

- **左侧控制台简化**：只保留数据导入入口
- **分析结果自动展示**：不需要用户手动选择分组维度
- **一键报告生成**：直接在结果下方显示"生成报告"按钮
- **删除冗余控制项**：不暴露 API 参数、分组切换等技术细节给普通用户

## 六、报告中心重新设计

### 6.1 设计目标

报告是产品的"最终交付物"，需要展示得更加正式和可用。

### 6.2 报告中心结构

```
┌─────────────────────────────────────────────────────────┐
│  选择数据源                                               │
│  [NHANES 样本] [CHARLS 样本] [我的数据]                  │
│                                                         │
│  机构名称：[        ] 报告标题：[              ]         │
│  阈值：10  每周承接：20 人                               │
│                                                         │
│  [生成报告] [下载 Markdown]                              │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  报告预览                                         │   │
│  │                                                   │   │
│  │  ## 示例机构心理健康风险识别报告                    │   │
│  │                                                   │   │
│  │  风险等级：[中等]  高风险占比：[XX%]               │   │
│  │                                                   │   │
│  │  - 重点人群：xxx                                  │   │
│  │  - 行动建议：xxx                                  │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 6.3 关键改动

- **报告预览更加正式**：去掉 JSON 和技术细节
- **支持一键下载**：Markdown 格式，方便直接使用
- **支持机构自定义**：机构名称、报告标题、阈值参数

## 九、设计原则

1. **用户任务优先**：每个页面都要回答"用户来这里要做什么"
2. **渐进式披露**：先展示结果，技术细节可展开查看
3. **减少认知负担**：不要让用户做选择，除非有必要
4. **正式感**：报告页面需要有机构正式报告的感觉
5. **移动端可用**：响应式设计，保证在平板/手机上也能展示
