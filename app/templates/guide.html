Getting Started

3 分钟看懂这个 API 怎么用。

这页专门给第一次接触产品的人准备。无论你是想先看页面效果，还是想直接调接口， 都可以从这里开始。

给产品用户

如果你不写代码，建议这样用。

Step 1

先看首页

快速理解平台能帮你识别什么风险、支持什么决策。

Step 2

打开在线演示

按年龄、收入、睡眠等维度切换查看重点人群。

Step 3

看报告中心

切换研究、管理、临床视角，直接拿去做汇报。

给开发者

如果你要接入接口，建议这样开始。

Step 1

读 `/docs`

先在交互文档里确认每个接口的参数和返回结构。

Step 2

调 `summary`

先拿总体摘要，确认服务状态和核心字段。

Step 3

按场景接入

再接画像、双周期对照、阈值模拟和简报接口。

推荐顺序

第一次使用，建议按这 5 个接口顺着看。

1. `health`

确认服务是否在线。

2. `summary`

查看当前风险、历史基线和核心摘要。

3. `population-profile`

查看不同人群的 PHQ-9 风险分布。

4. `cycle-comparison`

把当前风险和历史睡眠基线放在一起看。

5. `threshold-simulate`

估算筛查阈值变化后会筛出多少人。

最常用接口

先从这两个接口开始最稳。

/api/v1/summary /api/v1/cycle-comparison?group_by=age_band 
{{ summary_json }}

双周期示例

同一类人群，当前风险和历史背景怎么一起返回。

{{ comparison_json }}

复制即用

curl 示例

curl http://127.0.0.1:8000/api/v1/summary

curl "http://127.0.0.1:8000/api/v1/population-profile?group_by=income_band&min_participants=80"

curl "http://127.0.0.1:8000/api/v1/cycle-comparison?group_by=age_band"

curl "http://127.0.0.1:8000/api/v1/threshold-simulate?threshold=10&weekly_capacity=20"

前端接入

浏览器 `fetch` 示例

const summary = await fetch("/api/v1/summary").then((r) => r.json());

const profile = await fetch(
  "/api/v1/population-profile?group_by=sleep_band&min_participants=80"
).then((r) => r.json());

const comparison = await fetch(
  "/api/v1/cycle-comparison?group_by=income_band"
).then((r) => r.json());

你会得到什么

  • `summary`：总体风险、历史基线、样本覆盖和阈值参考。
  • `population-profile`：不同分组下的 PHQ-9 高风险占比。
  • `cycle-comparison`：同一维度下的当前风险与历史短睡眠背景。
  • `reports/{audience}`：按角色生成更容易汇报的文字摘要。

使用建议

  • 先看 `summary`，再看分组接口，不容易迷路。
  • 要给客户展示时，优先用首页、演示页和报告中心。
  • 要做系统接入时，优先用 `/docs` 和这页的复制示例。
  • 如果使用 `trycloudflare`，每次重启分享都要用新链接。
