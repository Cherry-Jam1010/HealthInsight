Getting Started

先做什么，再做什么，这一页讲清楚。

如果你是第一次使用 HealthInsight，不用自己猜顺序。下面已经按“产品用户”和“开发者”两条路径整理好了。

第一步

先判断你是哪种用户。

  • 如果你要给老师、客户或团队演示产品：走“产品用户路径”。
  • 如果你要接系统、调接口或做前端联调：走“开发者路径”。

关键文档

先把这几个文档收好。

下载快速开始 下载架构说明 下载 README 下载 OpenAPI JSON

产品用户路径

不写代码，就按这 3 步走。

第一步

打开首页

先理解平台能识别什么风险、能支持什么决策。

第二步

进入工作台

导入你自己的数据，或生成演示数据，查看分析结果。

第三步

打开报告中心

把同一份结果切换成研究、管理或临床能看懂的版本，下载 Markdown。

打开首页 进入工作台 打开报告中心

开发者路径

要接 API，就按这 4 步走。

第一步

打开 `/docs`

先确认所有接口、参数和返回结构。

第二步

先调 `summary`

先拿总体摘要，确认当前风险和历史基线字段。

第三步

再调分组接口

重点接 `population-profile` 和 `cycle-comparison`。

第四步

最后接模拟和简报

再接 `threshold-simulate` 和 `reports/{audience}`。

打开接口文档 下载 OpenAPI JSON

推荐顺序

第一次接 API，按这个顺序最不容易迷路。

1. `health`

确认服务在线。

2. `summary`

先读总体风险和历史基线。

3. `population-profile`

看不同人群的风险分布。

4. `cycle-comparison`

看当前风险和历史睡眠背景。

5. `threshold-simulate`

看阈值变化后的筛出量。

最先要调的接口

先调这两个。

  • 第一步：`/api/v1/summary`
  • 第二步：`/api/v1/cycle-comparison?group_by=age_band`

复制即用

命令行示例

curl http://127.0.0.1:8000/api/v1/health

curl http://127.0.0.1:8000/api/v1/summary

curl "http://127.0.0.1:8000/api/v1/population-profile?group_by=income_band&min_participants=80"

curl "http://127.0.0.1:8000/api/v1/cycle-comparison?group_by=age_band"

curl "http://127.0.0.1:8000/api/v1/threshold-simulate?threshold=10&weekly_capacity=20"

浏览器接入

`fetch` 示例

const summary = await fetch("/api/v1/summary").then((r) => r.json());

const profile = await fetch(
  "/api/v1/population-profile?group_by=sleep_band&min_participants=80"
).then((r) => r.json());

const comparison = await fetch(
  "/api/v1/cycle-comparison?group_by=income_band"
).then((r) => r.json());

你会拿到什么

  • `summary`：总体风险、历史基线、覆盖率和阈值参考。
  • `population-profile`：不同分组的 PHQ-9 高风险比例。
  • `cycle-comparison`：同一维度下的当前风险与历史短睡眠背景。
  • `reports/{audience}`：给不同角色的现成摘要语言。

最后提醒

  • 如果你是产品演示，先走：首页 → 演示页 → 报告中心。
  • 如果你是系统接入，先走：`/docs` → `summary` → 分组接口。
  • 如果你用 `trycloudflare`，每次重启分享都必须换成新链接。
  • 关键文档已经放在本页顶部，可以直接下载。
